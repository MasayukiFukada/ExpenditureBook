# 次世代出費管理システム（Next.js移行）開発計画

## 1. 目的
現在のJava/Spring Boot/MySQLベースのシステムを、Next.jsとドキュメントDB（lowDB）を用いたモダンなアーキテクチャに全面的にリプレースする。データ構造を「1ヶ月1ドキュメント」に最適化し、ユーザー体験とメンテナンス性を向上させる。

## 2. 技術スタック
- **Framework:** Next.js (App Router)
- **Language:** TypeScript
- **Styling:** Tailwind CSS + Shadcn UI (洗練されたUIのため)
- **Database:** lowDB (JSONベースのローカルデータベース)
  - ※ PoloDBは興味深い選択肢だが、依存関係の安定性を考慮し、まずはlowDBを優先する。
- **Charts:** Recharts または Chart.js
- **Icons:** Lucide React

## 3. データモデル（lowDB構造）
`migrated_data.json` の構造を継承し、以下のような構成とする。
- `db.json`: 1ヶ月単位のドキュメント配列（支出データ + 月次コメント）。
- **Category Master:** 
  - 案1: フロントエンドの定数として管理（変更頻度が低い場合、管理UI不要で最速）。
  - 案2: `categories.json` として別途管理し、バックエンド・フロントエンド両方から参照。
  - ※ ユーザー様の提案通り、まずはフロントエンドへの埋め込みを基本としつつ、インポート時のバリデーションにも利用可能な形を検討。

## 4. 開発フェーズ

### フェーズ1: プロジェクト基盤構築
- Next.jsプロジェクトの初期化とTypeScript設定。
- lowDBのセットアップと `migrated_data.json` からの初期データロード機能の実装。
- **一括インポート機能の基盤:** 外部JSONファイルを読み込み、DBを更新するAPI/機能の実装。

### フェーズ2: ダッシュボードと集計表示
- 月次・年次の支出集計ロジックの実装（事前に集計された `summary` フィールドの活用）。
- グラフ表示（月別推移、カテゴリ別内訳）の実装。
- 月間コメントの表示・編集機能。

### フェーズ3: 支出入力 UI のリプレース（重要）
- 「1日単位」で複数の支出項目を一括入力・編集できるフォームの開発。
- カレンダー形式または日別リスト形式での支出閲覧。
- カテゴリ選択のサジェスト機能（フロントエンド埋め込みのマスタを使用）。

### フェーズ4: 拡張機能とデータメンテナンス
- **高度なインポート/エクスポート:** JSON形式での一括バックアップ・リストア機能。
- データの整合性チェック。

## 5. 移行のポイント
- **カテゴリ管理の簡素化:** カテゴリをコード/定数として管理することで、DB参照を減らしレスポンスを向上させる。
- **データポータビリティ:** JSONベースの一括インポート機能を設けることで、将来の他環境への移行やバックアップを容易にする。

## 6. 検証・テスト計画
- `migrated_data.json` が新システムで正しく表示されるかの確認。
- 日次一括保存時のバリデーションテスト。
- 集計計算の正確性（旧システムの数値との照合）。
